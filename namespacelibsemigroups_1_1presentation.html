libsemigroups::presentation
Presentations

Defined in presentation.hpp.

This namespace contains various helper functions for the class Presentation. These functions could be functions of Presentation but they only use public member functions of Presentation, and so they are declared as free functions instead.

Adds the rule \(x^{-1}y^{-1}xy = id\) to p, after attempting to detect inverses from the rules in p, using try_detect_group_inverses. If id is UNDEFINED, then the right-hand side is the empty word.

Template Parameters

Parameters

xthe first word in the commutator.
ythe second word in the commutator.
idthe identity letter, or UNDEFINED for the empty word.
Exceptions
LibsemigroupsExceptionif x, y, or id contains a letter not belonging to p.alphabet(), if try_detect_group_inverses throws, or if x or y contains a letter for which no inverse was detected.

Adds the rule \(x^{-1}y^{-1}xy = id\) to p. The letter a with index i in inverses is the inverse of the letter in alphabet with index i. If id is UNDEFINED, then the right-hand side is the empty word.

Template Parameters
Wordthe type of the words in the presentation.
Parameters
pthe presentation.
xthe first word in the commutator.
ythe second word in the commutator.
alphabetthe alphabet, which should be a superset of the letters in x and y.
inversesthe inverses of the letters in alphabet.
idthe identity letter, or UNDEFINED for the empty word.
Exceptions
LibsemigroupsExceptionif alphabet, inverses, x, y, or id contains a letter not belonging to p.alphabet(); if alphabet contains duplicates; if inverses are not valid inverses for alphabet; or if x or y contains a letter not belonging to alphabet.
See also
throw_if_contains_duplicates, throw_if_bad_inverses throw_if_word_not_over_alphabet and throw_if_letter_not_in_alphabet.

template<typename Word>
void try_detect_group_inverses ( Presentation< Word > const & p,
Word & letters,
Word & inverses )

This function tries to deduce group theoretic inverses defined by the rules of the presentation p as following: the rules of the presentation where one side has length 2 and the other has length 0 are detected. For any such rule we remember that the first letter is a possible inverse of the second. If rules of the form ab=1 and ba=1 are detected, then a has inverse b and vice versa. If there are multiple different such rules and we deduce conflicting values for the inverse of a letter, then an exception is thrown.

Those letters where an inverse is detected are pushed into the back of the parameter letters, and the detected inverse is pushed into inverses. The parameters letters and inverses are modified in-place, and are not cleared before adding letters or their inverses.
